## 🔴 1. Immediate Bug Fixes & Refactors
These are broken paths, silent failures, and hardcoded values that violate the system architecture and should be fixed first.

*   **Fix `zot.py` and `cal.py` Ingestion Hooks:** Update `_trigger_ingest()` to use `$CLAUDE_HOME` instead of legacy `/home/ubuntu/.openclaw/workspace` or hardcoded `/home/hoanganhduc/` paths. Add `logging.warning` so missing ingestion scripts throw an error rather than failing silently.
*   **Remove Hardcoded User Configs:** Strip hardcoded user IDs (e.g., `webdav_user: "hoanganhduc"`) from Zotero/Calibre scripts and force them to load exclusively from `secrets.json`.
*   **Fix Environment Isolation in Bash Wrappers:** In scripts like `run_zot.sh`, change `WS="${OPENCLAW_WORKSPACE:-/workspace}"` to `WS="${CLAUDE_HOME:-$HOME/.claude}"` to stop the runtime from polluting the global filesystem.
*   **Update Stale Markdown Docs:** Do a regex sweep across `canonical/skills/**/*.md` to replace legacy `~/.codex/` and `~/.openclaw/` paths with the new standard `~/.claude/skills/_run.sh` entry points.

## 🏗️ 2. Architectural & Tool Improvements
These changes will make the tools safer, faster, and more extensible.

*   **SQLite Safety in `cal.py`:** Wrap all local SQLite database mutations in a programmatic file lock and auto-trigger a shadow backup (e.g., copy `metadata.db` to `/tmp/`) before executing updates.
*   **Bidirectional Citation Links:** Modify library ingest scripts to capture the returned `evidence_id` and automatically inject it into the Calibre/Zotero item's "Tags" or "Notes" field.
*   **Refactor `modal-research-compute`:** Rename and refactor this into a generic `remote-compute-provider` interface so researchers can swap Modal for AWS Batch, RunPod, or a local SLURM cluster without rewriting agent prompts.
*   **Optimize `docling` Ingestion:** 
    *   Add a fast pre-flight check (using `PyMuPDF` or `pdftotext`). Only boot the heavy PyTorch/VLM models if the fast parser fails or detects high image/formula density.
    *   Expose a `--remote-vlm-provider` flag to allow fallback to external APIs (like Azure/Anthropic) instead of hardcoding `ocrspace`.
*   **Standardize Document URIs:** Stop passing raw filepaths between agents. Implement a unified URI resolver (e.g., `doc://zotero/<id>` or `doc://calibre/<id>`) in the `run_skill.sh` wrapper.
*   **Lean Syntax Scanner Upgrade:** Update the `lean-strict-verification-gate` to detect and flag vacuous truths (e.g., unintentionally passing 0 = 1 => P) which currently bypass the syntax checker.

## 🔄 3. Autonomous Loop & Runtime Enhancements
These apply to `autonomous-research-loop-runtime` and system observability to prevent agents from getting stuck or losing data.

*   **Auto-Sync `recovery.md`:** Update the `append-iteration` command in the runtime helper to automatically write `recovery.md`. This prevents the JSON ledger (`iterations.jsonl`) and the markdown recovery file from ever drifting out of sync.
*   **Pre-Exhaustion Budget Warnings:** Modify the loop's `status` check to emit a warning (e.g., "80% budget consumed") to the LLM. Right now, it hits a hard wall; a warning lets the agent gracefully synthesize and exit.
*   **SageMath Timeout Partial Recovery:** Change the SageMath wrapper so that if a heavy computation times out, it returns the *highest tested bounds/state* instead of a total failure, allowing the agent to resume the search.
*   **Replace `session-logs` Bash Scripts:** Stop relying on brittle `rg` and `jq` bash pipelines to read agent memories. Build a dedicated Python command (`extract-transcript`) into the runtime helper to handle OS-agnostic quoting and error handling.
*   **Automate Self-Improvement Triggers:** Build a local error-hash registry into the runtime. If the exact same error happens 3 times, bypass the agent's ephemeral memory and programmatically inject a hard system prompt forcing the agent to propose a Canonical Integration Plan.
